# EPUB 人名注释版生成器 — 使用说明

## 换书只需要两步

### 第一步：创建角色映射文件

新建一个 JSON 文件（比如 `角色.json`），格式如下：

```json
[
  {
    "nick": "老卡",
    "colorClass": "ct-jose",
    "gen": "一代·卡系",
    "desc": "父亲，老流氓。",
    "origs": ["费多尔·巴夫洛维奇·卡拉马佐夫", "费多尔·巴夫洛维奇", "费多尔"]
  },
  {
    "nick": "二冷",
    "colorClass": "ct-orel",
    "gen": "二代·冷系",
    "desc": "次子，冷静理性，无神论者。",
    "origs": ["伊凡·费多罗维奇", "伊凡"]
  },
  {
    "nick": "小善",
    "colorClass": "ct-female",
    "gen": "二代·善系",
    "desc": "三子，善良虔诚，主角。",
    "origs": ["阿历克赛·费多罗维奇·卡拉马佐夫", "阿列克赛", "阿历克赛", "阿辽沙"]
  }
]
```

**字段说明：**
| 字段 | 含义 | 必填 |
|------|------|--------|
| `nick` | 中文绰号，显示在原名下方 | 是 |
| `colorClass` | 颜色分类，可选值：`ct-jose` `ct-orel` `ct-female` `ct-outsider` | 是 |
| `gen` | 代际/身份说明，显示在 tooltip 中 | 是 |
| `desc` | 一句话角色简介 | 是 |
| `origs` | epub 中实际出现的原名写法列表，**按字符长度从长到短排列** | 是 |

**关键规则：`origs` 必须从长到短排列！**

```
"origs": ["奥雷里亚诺·布恩迪亚上校", "奥雷里亚诺·布恩迪亚", "奥雷里亚诺"]
```

长名先匹配，被替换成占位符后，短名就不会误伤长名里的子串了。

### 第二步：运行脚本

```bash
python3 epub_annotator.py "你的书.epub" "角色.json" "输出.html"
```

三个参数：epub路径、角色映射JSON路径、输出HTML路径。

---

## 如果不知道书里人名怎么写

先提取 epub 里所有人名候选，再填 JSON：

```bash
python3 epub_annotator.py "你的书.epub" --extract-names
```

会输出所有带间隔号（`·`）的人名及其出现频率，按频率排序。用这个结果填 `origs`。

## 底层逻辑

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  1. EXTRACT  │ ──→ │  2. ANNOTATE  │ ──→ │  3. ASSEMBLE  │
│  解压epub    │     │  人名→ruby标签  │     │  填入HTML模板 │
│  提取段落    │     │  长名优先匹配   │     │  输出完整HTML │
│  提取脚注    │     │  占位符防误伤   │     │               │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 第一步：EXTRACT（提取）

epub 本质是 zip 文件，里面是 XHTML 文件。处理逻辑：

1. `zipfile` 打开 epub
2. 遍历所有 `.html`/`.xhtml` 文件
3. 用 HTML 解析器提取 `<p>` 标签里的纯文本（跳过 CSS class 为脚注列表的）
4. 每个文件 = 一个章节

### 第二步：ANNOTATE（注释）

这是核心算法——**长名优先 + 占位符防二次匹配**。

**问题：** "索菲亚·伊凡诺芙娜" 和 "伊凡" 共用子串 "伊凡"。如果先替换 "伊凡"→"二冷"，长名就变成 "索菲亚·二冷诺芙娜"。

**解法：**

```
原文: "伊凡·费多罗维奇去找索菲亚·伊凡诺芙娜"

替换表（按长度降序）:
  1. "伊凡·费多罗维奇"  → 二冷  (6字，先匹配)
  2. "索菲亚·伊凡诺芙娜" → 大妈二  (9字，先匹配)
  3. "伊凡"              → 二冷  (2字，最后匹配)

第一步: "伊凡·费多罗维奇去找索菲亚·伊凡诺芙娜"
       ↓ 匹配 "伊凡·费多罗维奇"，替换为占位符 \x00PH0\x00
       "\x00PH0\x00去找索菲亚·伊凡诺芙娜"

第二步: 匹配 "索菲亚·伊凡诺芙娜"，替换为占位符 \x00PH1\x00
       "\x00PH0\x00去找\x00PH1\x00"

第三步: 匹配 "伊凡" → 没有了！已经被前两步消耗掉了

第四步: 占位符替换回ruby标签
       "<ruby>二冷</ruby>去找<ruby>大妈二</ruby>"
```

### 第三步：ASSEMBLE（组装）

把注释后的段落内容插入 HTML 模板。模板包含：
- CSS 样式（ruby注释、侧边栏、tooltip、暗色模式）
- JavaScript 交互（悬停、点击锁定、简称/原名切换、目录跳转、滚动高亮）
- 侧边栏（目录、人物速查表、家族关系图）

模板在 `template.html`，不需要修改。动态内容通过占位符 `__CHAPTERS__` `__TOC__` `__CHIPS__` `__TREE__` 插入。
