## 用户场景与测试 *(必填)*

### 用户故事 1 - AI 服务商与模型选择 (优先级: P1)

用户进入设置页面，可以查看当前已配置的 AI 服务商列表。系统内置预设服务商（OpenAI、Anthropic、Google、DeepSeek、Qwen、Kimi、Doubao），用户选择后只需填入 API Key 即可使用。对于不在预设列表中的服务商，用户可以通过"自定义服务商"选项手动填写 OpenAI 兼容的 Base URL、API Key 和模型名称来接入任意 LLM 服务。用户选择一个服务商作为默认，并从可用模型列表中选择默认模型。切换后立即生效。

**优先级理由**: 选择服务商和模型是使用 AI 功能的前提条件，没有它就无法使用翻译、查词、摘要等核心功能。

**独立测试**: 可通过进入设置页、选择预设服务商、输入 API Key、选择模型、保存、然后触发翻译来完整测试。也可通过添加自定义服务商（填写 base URL + API Key + 模型名）来验证非预设服务商的接入。

**验收场景**:

1. **假设** 用户首次进入设置页面，**当** 页面加载完成，**则** 显示内置预设服务商列表（OpenAI、Anthropic、Google、DeepSeek、Qwen、Kimi、Doubao）以及"自定义服务商"入口，已配置 API Key 的服务商高亮标记，并展示当前默认服务商和默认模型
2. **假设** 用户选择了一个预设服务商（如 DeepSeek），**当** 选择完成，**则** 展示该服务商的可用模型列表（从其 API 动态获取或使用内置默认列表），默认选中推荐模型
3. **假设** 用户选择"自定义服务商"，**当** 展开配置表单，**则** 显示 Base URL 输入框、API Key 输入框、模型名称输入框，填写完成后可验证和保存
4. **假设** 用户选择了新模型并保存，**当** 保存成功，**则** 显示保存成功提示，且后续 AI 功能使用新模型

---

### 用户故事 2 - API Key 管理 (优先级: P1)

用户可以在设置页面中输入和管理各 AI 服务商的 API Key。对于预设服务商，输入 API Key 后系统自动调用其 /models 端点验证有效性。对于自定义服务商，系统通过 OpenAI 兼容协议的 /v1/models 端点验证。验证通过的 Key 被安全存储。用户可以随时更新或删除已保存的 Key。

**优先级理由**: 没有 API Key 就无法调用任何 AI 服务，这是所有 AI 功能的基础依赖。

**独立测试**: 可通过输入 API Key、触发验证、确认验证结果、执行 AI 调用来完整测试。

**验收场景**:

1. **假设** 用户选择了 OpenAI 服务商，**当** 输入有效的 API Key 并点击验证，**则** 显示验证成功，Key 被标记为已验证状态
2. **假设** 用户选择了 DeepSeek 服务商，**当** 输入 API Key 并点击验证，**则** 系统调用 DeepSeek 的 OpenAI 兼容端点验证 Key 有效性
3. **假设** 用户配置了自定义服务商，**当** 填写 base URL 和 API Key 并点击验证，**则** 系统调用 `{base_url}/v1/models` 验证连通性
4. **假设** 用户输入了无效的 API Key，**当** 点击验证，**则** 显示"Key 无效"错误提示
5. **假设** 用户已保存了一个 API Key，**当** 进入设置页面，**则** Key 以掩码形式显示（如 `sk-****abcd`），不暴露完整值
6. **假设** 用户删除了某个服务商的 API Key，**当** 删除完成，**则** 该服务商标记为未配置状态，且若该服务商为当前默认则提示用户切换

---

### 用户故事 3 - 提示词配置 (优先级: P2)

用户可以自定义翻译、查词解释、章节摘要等 AI 功能的系统提示词（System Prompt），从而控制 AI 的输出风格和格式。系统提供专业级默认提示词模板，这些模板经过精心设计，包含明确的输出格式约束、角色定义、翻译原则和边界条件。用户可以在默认基础上修改，也可以一键恢复。

**优先级理由**: 专业的默认提示词是 AI 输出质量的基础保障；自定义能力则为高级用户提供精细化控制。

**独立测试**: 可通过修改翻译提示词、保存、触发翻译、观察输出格式变化来完整测试。

**验收场景**:

1. **假设** 用户进入提示词配置区域，**当** 页面加载，**则** 显示翻译、查词、摘要三个提示词编辑区，各填充专业级默认模板
2. **假设** 用户使用默认提示词触发翻译，**当** 翻译完成，**则** 输出格式规范、人名保留原文、风格忠实于原文、不包含多余解释
3. **假设** 用户修改翻译提示词并保存，**当** 触发翻译功能，**则** 翻译结果遵循自定义提示词的指令风格
4. **假设** 用户点击"恢复默认"按钮，**当** 操作完成，**则** 该提示词恢复为系统内置的专业级模板

---

### 边界情况

- 当用户切换服务商但未配置 API Key 时，系统应提示先配置 Key
- 当所有服务商均未配置 API Key 时，AI 相关功能应显示"请先在设置中配置 AI 服务"的引导提示
- 当 AI 服务调用因 Key 过期或额度不足失败时，设置页应展示对应服务商的警告状态
- 当网络不可用时，API Key 验证应给出明确的离线提示，而非无响应
- 支持用户同时配置多个服务商的 Key，方便随时切换
- 自定义服务商的 Base URL 格式不正确时（缺少协议前缀、无法解析等），应给出明确的格式错误提示
- 自定义服务商的 API 端点不兼容 OpenAI 协议时，验证应返回有意义的错误信息
- 用户可添加多个自定义服务商，每个自定义服务商有独立的配置

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系统必须提供独立的设置页面入口，可在全局导航中直接访问
- **FR-002**: 系统必须内置预设服务商列表（OpenAI、Anthropic、Google、DeepSeek、Qwen、Kimi、Doubao），每个预设包含服务商名称、API 端点和默认模型列表
- **FR-003**: 系统必须支持自定义服务商，允许用户填写 OpenAI 兼容的 Base URL、API Key 和模型名称
- **FR-004**: 系统必须允许用户为每个服务商输入和保存 API Key
- **FR-005**: 系统必须在保存 API Key 前验证其有效性（预设服务商调用各自 API 端点，自定义服务商调用 `{base_url}/v1/models`）
- **FR-006**: 系统必须以掩码形式展示已保存的 API Key（仅显示首尾字符）
- **FR-007**: 系统必须允许用户选择默认 AI 服务商和默认模型
- **FR-008**: 系统必须支持自定义翻译、查词、摘要功能的系统提示词，并提供专业级默认模板
- **FR-009**: 默认翻译提示词必须包含：角色定义、翻译原则（信达雅）、人名保留规则、输出格式约束（纯译文无解释）
- **FR-010**: 默认查词提示词必须包含：JSON 格式输出约束、词性标注、音标标注、上下文感知释义
- **FR-011**: 默认摘要提示词必须包含：角色定义（文学评论家视角）、输出长度约束（2-3 句话）、内容覆盖要求（情节要点+关键转折）
- **FR-012**: 系统必须为所有可调参数提供"恢复默认值"功能
- **FR-013**: 系统必须在 API Key 验证成功或失败后给出明确的反馈提示
- **FR-014**: 系统必须将所有设置持久化存储，刷新页面后配置不丢失
- **FR-015**: 系统必须在未配置任何 AI 服务时，在 AI 功能入口处展示引导提示跳转到设置页

### 关键实体

- **AI 配置 (Settings)**: 存储用户的全局 AI 偏好，包括默认服务商、默认模型
- **服务商凭证 (Provider Credential)**: 每个服务商对应的 API Key 及其验证状态（已验证/未验证/验证失败）；自定义服务商额外存储 Base URL
- **预设服务商 (Provider Preset)**: 系统内置的服务商配置，包含名称、API 端点、默认模型列表、认证方式
- **自定义服务商 (Custom Provider)**: 用户手动添加的 OpenAI 兼容服务商，包含自定义名称、Base URL、API Key、模型名称
- **提示词模板 (Prompt Template)**: 翻译、查词、摘要三类 AI 功能各自对应的系统提示词（专业级默认模板）

## 假设

- 本应用为单用户桌面/本地部署应用，无需多用户权限隔离
- API Key 存储在本地数据库中，不涉及云端密钥管理服务
- 预设服务商（DeepSeek、Qwen、Kimi、Doubao）均提供 OpenAI 兼容 API，可通过统一的 `{base_url}/v1/models` 和 `{base_url}/v1/chat/completions` 端点调用
- 预设服务商的 API 端点参考：DeepSeek（`https://api.deepseek.com`）、Qwen（`https://dashscope.aliyuncs.com/compatible-mode`）、Kimi（`https://api.moonshot.cn/v1`）、Doubao（`https://ark.cn-beijing.volces.com/api/v3`）
- Anthropic 不兼容 OpenAI 协议，需通过 `@ai-sdk/anthropic` 专用 SDK 调用，验证端点为 `https://api.anthropic.com/v1/models`
- 现有通过环境变量配置 AI 的方式仍作为兜底方案保留（设置页配置优先级更高）
- 翻译、查词、摘要等现有 AI 功能调用时，优先读取设置页的用户配置，环境变量作为 fallback
- 温度和最大 Token 等生成参数使用系统默认值，不暴露给用户配置
